# Med Marking Services

Система интеграции медицинской организации с ГИС МТ «Честный ЗНАК»
Разработано в рамках выпускной квалификационной работы (ВКР).

Проект автоматизирует учёт и выбытие маркированных медицинских изделий на базе оперативного складского учёта частной клиники. Включает фронтенд для медперсонала, BFF-слой, адаптер к «больничной информационной системе» (имитация) и коннектор к API «Честного ЗНАКа» с реализацией паттерна Saga для надёжной отправки данных о выводе из оборота.

## Архитектура

- **frontend** – React + TypeScript, Material UI. Две роли: медсестра (создание операций) и старшая медсестра (утверждение).
- **bff** – Backend For Frontend (FastAPI). Проксирует запросы, агрегирует данные из HIS Adapter и Connector.
- **his-adapter** – эмуляция медицинской информационной системы (SQLite). Хранит остатки, SGTIN, документы. После утверждения документа вызывает Connector.
- **gis-mt-connector** – реализует Saga-оркестрацию вывода из оборота. Получает список КИ, подписывает документ через локальный сервис подписания, отправляет в True API, опрашивает статус.
- **local-signer-stub / local-signer** – заглушка для подписания (или реальный адаптер для КриптоПро).

Все сервисы упакованы в Docker Compose.

## Методическая основа и возможности доработки

Разработанная система может служить методической основой для интеграции с произвольной медицинской информационной системой. Модуль `his-adapter` реализован как сменный компонент, что позволяет адаптировать проект к любой реляционной базе данных (PostgreSQL, MySQL, Oracle и др.) путём замены уровня доступа к данным. Архитектура спроектирована так, чтобы её можно было расширять: подключать реальные API МИС, добавлять новые типы документов, реализовывать автоматическое считывание кодов маркировки и развивать аналитическую отчётность без изменения основных компонентов.

## Быстрый старт (локальный запуск)

### Требования
- Docker и Docker Compose
- Git
- (опционально) Python 3.10+ для ручного запуска

### Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/lyamancini-dev/med-marking-services-public.git
cd med-marking-services-public
```

### Шаг 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта (рядом с `docker-compose.yml`). Минимальный пример:

```ini
# URL локального сервиса подписания (используется заглушка)
SIGNER_URL=http://local-signer-stub:8000/sign

# Адреса песочницы Честного ЗНАКа
TRUE_API_BASE_URL=https://markirovka.sandbox.crptech.ru/api/v3/true-api
TRUE_API_BASE_URL_V4=https://markirovka.sandbox.crptech.ru/api/v4/true-api

# ИНН участника (тестовый)
PARTICIPANT_INN=1234567890

# Товарная группа (для медицинских изделий)
PRODUCT_GROUP=wheelchairs
```

> **Важно:** Для реального взаимодействия с ГИС МТ нужно указать боевые URL, корректный ИНН и использовать `local-signer` вместо `local-signer-stub` (с настроенным КриптоПро и сертификатом). В публичном репозитории приведены только заглушки.

### Шаг 3. Запуск всех сервисов

```bash
docker-compose up --build
```

После сборки и запуска будут доступны:

| Сервис           | Порт | URL                         |
|-----------------|------|-----------------------------|
| Frontend        | 3000 | http://localhost:3000       |
| BFF API         | 8002 | http://localhost:8002/health |
| HIS Adapter API | 8000 | http://localhost:8000       |
| GIS MT Connector| 8001 | http://localhost:8001       |

### Шаг 4. Использование

- Откройте `http://localhost:3000` в браузере.
- Выберите роль **Медсестра** или **Старшая м/с** в верхнем меню.
- **Медсестра** может:
  - Просматривать остатки на складах.
  - Создавать черновики **перемещения** или **списания** (для маркированных товаров нужно вручную вводить SGTIN).
- **Старшая медсестра** дополнительно может утверждать документы в журнале. После утверждения списания запускается Saga отправки в ГИС МТ (статус отслеживается в журнале).

## Ручной запуск (без Docker)

Каждый сервис можно запустить отдельно, используя Python (рекомендуется virtualenv).  
Установите зависимости из `requirements.txt` внутри каждого сервиса и запустите:

```bash
# Пример для BFF
cd services/bff
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8002
```

Аналогично для `his-adapter` (порт 8000) и `gis-mt-connector` (порт 8001).  
Фронтенд запускается стандартно:

```bash
cd frontend
npm install
npm start
```

## Тестовые данные

В `his-adapter` при первом запуске автоматически создаются:

- **Склады**: «Центральный склад», «Кабинет косметологии 1», «Операционная» и др.
- **Товары** (немаркированные и маркированные):
  - Шприц 5 мл (немарк., остаток 100)
  - Эндопротез тазобедренный (марк., GTIN 04601234567890, остаток 5)
  - Катетер уретральный (марк., остаток 3)
- **SGTIN** для маркированных товаров (тестовые строки с символом-разделителем `\x1d`).

Для демонстрации списания можно выбрать маркированный товар, ввести количество и отсканировать/ввести тестовый SGTIN из списка (например, `010460123456789021serial1\x1d93crypto`).

## Демо-режим подписания

По умолчанию используется **stub** – он не требует реальной ЭЦП и просто возвращает хеш данных в base64.  
Если у вас есть настроенный КриптоПро и сертификат, замените в `docker-compose.yml`:

```yaml
SIGNER_URL=http://local-signer:8003/sign   # реальный сервис
```

И соберите образ `local-signer` (папка `services/local-signer`). В коде нужно указать актуальный отпечаток сертификата через переменную окружения `CERT_THUMBPRINT`.
